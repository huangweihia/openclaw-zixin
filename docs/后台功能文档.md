# OpenClaw 智信 — 管理后台功能文档

> 本文档按**当前侧边栏菜单**整理，与 `resources/js/admin/constants/menu.js` 及 `resources/js/admin/router/index.js` 保持一致。  
> 访问入口一般为 **`/admin/`**（具体以前端部署与 `vite` 配置为准）。  
> 菜单可见性受 **`admin_users` 权限点**控制：无对应 `read` 权限的项不会出现在侧边栏；接口层另有 `perm` 中间件校验，避免「菜单隐藏仍可调 API」。

---

## 1. 通用说明

| 项目 | 说明 |
|------|------|
| 前端技术 | Vue 3 + Vue Router + Element Plus + Axios |
| 页面框架 | 列表/运营页普遍使用 `AdminPageShell`（标题、副标题、操作区、工具栏）+ `AdminCard` + `AdminPagination` |
| 认证 | 登录后通过 Sanctum 会话访问 `/api/admin/*`；未登录访问受保护路由会跳转登录页 |
| 超级管理员 | `admin_permissions` 含 `*` 时视为拥有全部菜单与接口权限 |

### 1.1 不在侧边栏但属于后台的路由

| 路由示例 | 说明 |
|----------|------|
| `/admin/login` | 登录页 |
| `/admin/users/:id` | 用户详情/编辑（从「用户管理」进入） |
| `/admin/articles/create`、`/admin/articles/:id/edit` | 新建/编辑文章 |
| `/admin/projects/create`、`/admin/projects/:id/edit` | 新建/编辑项目 |

| 权限点 | `admin:settings:read`（与「系统与站点」同级权限点） |

- 前台入口是否展示由「启用」开关与题库数据是否就绪共同决定（与首页模块一致）。
