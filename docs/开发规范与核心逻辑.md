## 🎨 二、开发规范

### 2.1 低耦合原则（⚠️ 核心原则）

**为什么使用组件库？**

1. **低耦合**：组件独立，修改不影响其他页面
2. **可复用**：一次开发，多处使用
3. **易维护**：出问题只改组件，不用改每个页面
4. **统一性**：所有页面风格一致

**错误示例（❌ 高耦合）：**
```blade
<!-- 每个页面都写一遍相同的样式 -->
<div style="background: white; color: black; padding: 20px;">
    内容
</div>

<!-- 另一个页面又写一遍 -->
<div style="background: white; color: black; padding: 20px;">
    内容
</div>
```

**正确示例（✅ 低耦合）：**
```blade
<!-- 使用组件 -->
<x-card>
    内容
</x-card>

<!-- 另一个页面也用组件 -->
<x-card>
    内容
</x-card>
```

**修改时：**
- ❌ 高耦合：需要改每个页面的样式
- ✅ 低耦合：只改 `components/card.blade.php` 一个文件

---

### 2.2 换肤兼容原则（⚠️ 核心原则）

**所有颜色必须使用 CSS 变量！**

**错误示例（❌ 换肤后文字看不见）：**
```blade
<!-- 硬编码白色背景 -->
<div style="background: #ffffff; color: #000000;">
    内容
</div>

<!-- 硬编码黑色背景 -->
<div style="background: #000000; color: #ffffff;">
    内容
</div>
```

**正确示例（✅ 自动跟随主题）：**
```blade
<!-- 使用 CSS 变量 -->
<div style="background: var(--bg-card); color: var(--text);">
    内容
</div>

<!-- 使用组件类（组件内部已使用 CSS 变量） -->
<div class="oc-card">
    内容
</div>
```

**CSS 变量定义（skins.css）：**
```css
/* 浅色皮肤 */
:root[data-skin="style1"] {
    --bg-card: #ffffff;
    --text: #0f172a;
}

/* 深色皮肤 */
:root[data-skin="style5"] {
    --bg-card: #1a1a1a;
    --text: #f1f5f9;
}
```

**换肤时：**
- ❌ 硬编码：需要改每个页面的颜色
- ✅ CSS 变量：自动切换，无需改页面

## 📐 三、开发流程

### 3.1 正确的开发顺序（⚠️ 必须严格遵守）

**拿到功能清单后，按以下步骤开发：**

```
第 1 步：阅读功能清单 → 了解功能做什么
        ↓
第 2 步：查看【涉及数据库】→ 知道用哪些表
        ↓
第 3 步：查看 migrations 或 02-数据库表字段详细设计.md → 了解字段结构
        ↓
第 4 步：编写后端接口（控制器 + 路由）→ 实现数据逻辑
        ↓
第 5 步：查看【交互详情】+【数据逻辑】→ 理解交互原理
        ↓
第 6 步：查看原型图 → 找到对应页面的原型
        ↓
第 7 步：开发页面（Blade 视图）→ 对照原型图 1:1 还原
        ↓
第 8 步：自测功能闭环 → 确保能正常使用
        ↓
第 9 步：更新功能清单状态为 ✅ 已完成
```

### 3.2 每一步的详细说明

#### 第 1 步：阅读功能清单

**目的：** 了解这个功能要做什么

**操作：**
1. 打开 `docs/功能清单/XX-模块名.md`
2. 阅读每个功能的**功能描述**
3. 理解功能的业务逻辑

**示例：**
```markdown
## 功能 6：实时动态滚动
**功能描述：** 实时动态自动滚动显示
**页面位置：** 首页实时动态区
```
→ 理解：需要在首页显示一个自动滚动的动态列表

---

#### 第 2 步：查看涉及数据库

**目的：** 知道接口需要操作哪些表

**操作：**
1. 查看功能清单中的**涉及数据库**字段
2. 记录需要用到的表名和字段

**示例：**
```markdown
**涉及数据库：** 表：users, subscriptions
              字段：users.name, users.role, subscriptions.plan
```
→ 理解：需要查询 `users` 和 `subscriptions` 表，获取用户姓名、角色、套餐信息

---

#### 第 3 步：查看数据库设计文档

**目的：** 了解表结构，正确编写查询

**操作：**
1. 打开 `docs/02-数据库表字段详细设计.md`
2. 查找对应表的详细设计
3. 或者查看 `database/migrations/*_create_xxx_table.php`

**示例：**
```sql
-- users 表
users.name VARCHAR(255) -- 用户昵称
users.role ENUM('user', 'vip', 'svip', 'admin') -- 用户角色

-- subscriptions 表
subscriptions.plan ENUM('free', 'vip', 'svip') -- 套餐类型
```
→ 理解：`role` 和 `plan` 是枚举类型，查询时要注意

---

#### 第 4 步：编写后端接口

**目的：** 实现数据逻辑，为页面提供数据

**操作：**
1. 创建/编辑控制器 `app/Http/Controllers/XxxController.php`
2. 编写方法，实现**数据逻辑**中的要求
3. 添加路由 `routes/web.php`

**示例（实时动态）：**
```php
// HomeController.php
public function index()
{
    // 数据逻辑：查询最近 10 条用户开通 VIP 记录
    $vipActivities = DB::table('subscriptions')
        ->join('users', 'subscriptions.user_id', '=', 'users.id')
        ->where('subscriptions.plan', '!=', 'free')
        ->orderByDesc('subscriptions.created_at')
        ->limit(10)
        ->get(['users.name', 'users.role', 'subscriptions.plan', 'subscriptions.created_at']);
    
    return view('home', compact('vipActivities'));
}
```

---

#### 第 5 步：查看交互详情和数据逻辑

**目的：** 理解交互细节，完善接口

**操作：**
1. 回到功能清单，仔细阅读**交互详情**表格
2. 阅读**数据逻辑**的每一步
3. 确保接口实现了所有逻辑

**示例：**
```markdown
**交互详情：**
| 元素 | 点击/悬停 | 反应 | 动作 |
|------|----------|------|------|
| 实时动态 | 悬停 | 暂停滚动 | 无 |
| 实时动态 | 点击 | 无 | 无（纯展示）|

**数据逻辑：**
1. 查询最近 10 条用户开通 VIP 记录
2. 对用户手机号/邮箱脱敏处理
3. 每 5 秒自动滚动一条
```
→ 理解：
- 接口需要返回脱敏的数据
- 前端需要实现悬停暂停、5 秒滚动

---

#### 第 6 步：查看原型图

**目的：** 了解页面长什么样，确保 1:1 还原

**操作：**
1. 打开 `docs/原型图/XX-页面名.md`
2. 查看页面布局、组件使用、交互说明
3. 记录需要使用的组件

**示例：**
```markdown
## 页面布局
```
┌─────────────────────────────────────────┐
│  @component-top-nav（顶部导航栏）       │
├─────────────────────────────────────────┤
│  实时动态区                             │
│  ┌─────────────────────────────────┐   │
│  │ 动态列表（自动滚动）             │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

**使用组件：** @component-top-nav, @component-footer
```
→ 理解：
- 使用 `<x-top-nav />` 组件
- 实时动态区需要自动滚动效果

---

#### 第 7 步：开发页面

**目的：** 实现页面，对照原型图 1:1 还原

**操作：**
1. 创建/编辑视图 `resources/views/xxx.blade.php`
2. 使用组件库的组件（`<x-xxx />`）
3. 使用 CSS 变量（`var(--xxx)`）
4. 调用第 4 步编写的接口获取数据
5. 实现第 5 步的交互逻辑

**示例：**
```blade
@extends('layouts.app')

@section('content')
    <x-top-nav :user="auth()->user()" />
    
    <section class="home-section">
        <h2>实时动态</h2>
        <div class="realtime-dynamic" id="dynamicList">
            @foreach($vipActivities as $activity)
                <div class="dynamic-item">
                    <span>{{ Str::limit($activity->name, 1, '***') }}</span>
                    <span>开通了{{ $activity->plan }}会员</span>
                </div>
            @endforeach
        </div>
    </section>
@endsection

@push('scripts')
<script>
// 交互：悬停暂停，5 秒滚动
const list = document.getElementById('dynamicList');
let timer = setInterval(scrollNext, 5000);
list.addEventListener('mouseenter', () => clearInterval(timer));
list.addEventListener('mouseleave', () => timer = setInterval(scrollNext, 5000));
</script>
@endpush
```

### 3.3 常见错误与避免方法

#### ❌ 错误 1：先写页面再补接口

**错误做法：**
1. 先看原型图，写页面
2. 发现需要数据，再写接口
3. 接口字段和页面不匹配，返工

**正确做法：**
1. 先看数据库，写接口
2. 再看原型图，写页面
3. 页面调用接口，一次成功

---

#### ❌ 错误 2：不看数据库设计

**错误做法：**
- 不知道字段类型，接口返回错误格式
- 不知道关联关系，查询效率低

**正确做法：**
- 先查 `02-数据库表字段详细设计.md`
- 了解字段类型、关联关系、索引
- 编写正确的查询语句

---

#### ❌ 错误 3：不看交互详情

**错误做法：**
- 只实现静态页面，没有交互
- 漏掉悬停、点击、验证等逻辑

**正确做法：**
- 仔细阅读**交互详情**表格
- 实现所有交互（悬停、点击、验证等）
- 自测所有交互是否正常

---

#### ❌ 错误 4：不对照原型图

**错误做法：**
- 自己发挥，页面和原型图不一致
- 组件使用错误，样式不统一

**正确做法：**
- 打开原型图文档，对照开发
- 使用指定的组件（`@component-xxx`）
- 布局、颜色、间距与原型图一致

---

#### ❌ 错误 5：不更新功能清单

**错误做法：**
- 开发完不更新状态
- 其他人不知道功能是否完成

**正确做法：**
- 开发完立即更新**开发状态**为 ✅
- 测试完更新**测试状态**
- 创建模块完成记录文档

---

#### ❌ 错误 6：硬编码颜色（换肤后文字看不见）

**错误做法：**
```blade
<div style="background: white; color: black;">
    内容
</div>
```

**正确做法：**
```blade
<div class="oc-card">
    内容
</div>
```

---

#### ❌ 错误 7：重复造轮子（不用组件库）

**错误做法：**
```blade
<!-- 自己写输入框 -->
<input style="padding: 10px; border: 1px solid #ddd;" />
```

**正确做法：**
```blade
<!-- 使用组件 -->
<input class="oc-input" />
```

## 🔑 四、核心逻辑

### 4.1 用户认证

- **注册**：`POST /register`，邮箱 + 验证码 + 密码
- **登录**：`POST /login`，邮箱 + 密码，支持 remember_token
- **登出**：`POST /logout`，清除 Session
- **验证码**：`POST /register/send-code`，SMTP 发送，5 分钟有效

### 4.2 VIP 系统

- **用户角色**：`users.role` = `user` | `vip` | `svip` | `admin`
- **VIP 到期**：`users.subscription_ends_at`
- **VIP 标识**：`<x-badge variant="warning">👑 VIP</x-badge>`

### 4.3 内容系统

- **文章**：`articles` 表，支持 VIP 锁定
- **项目**：`projects` 表，GitHub 项目展示
- **评论**：`comments` 表，多态关联（`commentable_type`）
- **点赞**：`user_actions` 表，多态关联
- **收藏**：`favorites` 表，多态关联

### 4.4 互动系统

- **点赞**：`POST /articles/{id}/like`，返回 JSON
- **收藏**：`POST /articles/{id}/favorite`，返回 JSON
- **评论**：`POST /articles/{id}/comments`，支持回复（`parent_id`）
- **举报**：`POST /comments/{id}/report`，返回 JSON

### 4.5 支付系统

- **创建订单**：`POST /payments/orders`
- **支付结果**：`GET /payments/result?order_no=`
- **演示支付**：`POST /payments/orders/{order}/simulate-paid`
