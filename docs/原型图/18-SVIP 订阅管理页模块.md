## 📋 三、交互说明表

| 区域 | 元素 | 点击/悬停 | 反应 | 动作 |
|------|------|----------|------|------|
| 导航栏 | Logo（🦀） | 点击 | 无 | 刷新首页 |
| 创建订阅 | 按钮 | 点击 | 打开弹窗 | 无 |
| 订阅卡片 | 卡片 | 悬停 | 上移 4px | 无 |
| 订阅卡片 | 编辑 | 点击 | 打开编辑弹窗 | 无 |
| 订阅卡片 | 删除 | 点击 | 弹出确认框 | DELETE /api/subscriptions/{id} |
| 订阅卡片 | 暂停/启用 | 点击 | 切换状态 | PUT /api/subscriptions/{id}/toggle |

## 🔌 四、API 接口

### 创建订阅接口

**请求：**
```http
POST /api/svip/subscriptions
Content-Type: application/json

{
  "name": "竞品监控",
  "keywords": ["AI 副业", "变现", "案例"],
  "exclude_keywords": ["招聘", "广告"],
  "sources": ["github", "zhihu", "juejin"],
  "frequency": "daily",
  "push_methods": ["email", "wechat"]
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "竞品监控",
    "is_active": true,
    "created_at": "2026-04-03 10:00:00"
  },
  "message": "订阅创建成功"
}
```

## 📱 五、响应式规则

```
桌面端（>= 1024px）：
- 订阅卡片：单列
- 数据源：多列布局
- 按钮：横向布局

平板端（768px - 1023px）：
- 订阅卡片：单列
- 数据源：2 列布局
- 按钮：横向布局

手机端（< 768px）：
- 订阅卡片：单列
- 数据源：单列布局
- 按钮：纵向布局
```
