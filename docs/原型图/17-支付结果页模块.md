## 🔌 四、API 接口

### 支付结果查询接口

**请求：**
```http
GET /api/payments/order/{order_no}/status
```

**响应（成功）：**
```json
{
  "success": true,
  "data": {
    "order_no": "20260403100001",
    "status": "paid",
    "amount": 99.00,
    "paid_at": "2026-04-03 10:05:00",
    "vip_expires_at": "2027-04-03 10:05:00",
    "plan": "vip"
  }
}
```

**响应（失败）：**
```json
{
  "success": false,
  "data": {
    "order_no": "20260403100001",
    "status": "failed",
    "error_message": "支付失败，请重试"
  }
}
```

## 🛡️ 六、安全说明

### 支付安全

1. **HTTPS 加密** - 所有支付请求使用 HTTPS 加密
2. **订单签名** - 订单数据签名防篡改
3. **支付回调验证** - 验证支付回调签名
4. **幂等性** - 支付回调支持幂等处理

### 数据一致性

1. **事务处理** - 支付成功和 VIP 开通在同一事务
2. **订单状态** - 订单状态不可逆（paid→refunded）
3. **日志记录** - 完整记录支付日志
