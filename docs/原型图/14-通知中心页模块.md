## 🎨 二、组件使用详情

### 组件 1：通知列表

**使用组件：** @component-notification-list + @component-badge

**布局结构：**
```
┌─────────────────────────────────────────┐
│  🔴  您的文章已通过审核                 │
│  ─────────────────────────────────────  │
│  2026-04-03 10:30  |  [查看详情]       │
└─────────────────────────────────────────┘
```

**组件属性：**
- 未读通知：红色圆点 + 白色背景
- 已读通知：无圆点 + 灰色背景
- 悬停：背景变色

**交互说明：**

| 元素 | 点击/悬停 | 反应 | 动作 |
|------|----------|------|------|
| 通知卡片 | 悬停 | 背景变色 | 无 |
| 通知卡片 | 点击 | 标记为已读 | 跳转对应页面 |
| 查看详情 | 点击 | 标记为已读 | 跳转对应页面 |
| 立即续费 | 点击 | 无 | 跳转 VIP 页 |

---

### 组件 2：通知类型筛选

**使用组件：** @component-select

**布局结构：**
```
┌─────────────────────────────────────────┐
│  [全部通知▼]                            │
│  ─────────────────────────────────────  │
│  全部通知                               │
│  审核通知                               │
│  互动通知                               │
│  会员通知                               │
│  系统公告                               │
└─────────────────────────────────────────┘
```

**交互说明：**

| 元素 | 点击/悬停 | 反应 | 动作 |
|------|----------|------|------|
| 类型下拉 | 点击 | 展开列表 | 无 |
| 类型选项 | 点击 | 选中类型 | 刷新列表 |

---

### 组件 3：批量操作

**使用组件：** @component-button（次按钮）

**布局结构：**
```
┌─────────────────────────────────────────┐
│  [全部已读]  [清空通知]                 │
└─────────────────────────────────────────┘
```

**交互说明：**

| 元素 | 点击/悬停 | 反应 | 动作 |
|------|----------|------|------|
| 全部已读 | 点击 | 弹出确认框 | PUT /api/notifications/read-all |
| 清空通知 | 点击 | 弹出确认框 | DELETE /api/notifications/clear |

## 📋 三、交互说明表

| 区域 | 元素 | 点击/悬停 | 反应 | 动作 |
|------|------|----------|------|------|
| 导航栏 | Logo（🦀） | 点击 | 无 | 刷新首页 |
| 导航栏 | 通知图标 | 点击 | 红点消失 | 跳转通知中心 |
| 筛选器 | 类型下拉 | 点击 | 展开列表 | 无 |
| 筛选器 | 搜索框 | 输入 | 实时建议 | GET /api/notifications/suggest |
| 通知卡片 | 卡片 | 悬停 | 背景变色 | 无 |
| 通知卡片 | 卡片 | 点击 | 标记已读 | 跳转对应页面 |
| 全部已读 | 按钮 | 点击 | 弹出确认框 | PUT /api/notifications/read-all |
| 清空通知 | 按钮 | 点击 | 弹出确认框 | DELETE /api/notifications/clear |
| 分页器 | 页码 | 点击 | 高亮当前页 | 加载对应页数据 |

## 🔌 四、API 接口

### 通知列表接口

**请求：**
```http
GET /api/notifications?page=1&per_page=20&type=all
```

**响应：**
```json
{
  "success": true,
  "data": {
    "notifications": [
      {
        "id": 1,
        "type": "audit_approved",
        "title": "审核通过",
        "content": "您的文章《AI 副业案例》已通过审核",
        "is_read": false,
        "link": "/dashboard/publishes/1",
        "created_at": "2026-04-03 10:30:00"
      },
      {
        "id": 2,
        "type": "like_received",
        "title": "获得点赞",
        "content": "张三 点赞了您的文章",
        "is_read": false,
        "link": "/articles/1",
        "created_at": "2026-04-03 10:00:00"
      }
    ],
    "unread_count": 5,
    "pagination": {
      "total": 50,
      "per_page": 20,
      "current_page": 1,
      "last_page": 3
    }
  }
}
```

### 标记全部已读接口

**请求：**
```http
PUT /api/notifications/read-all
```

**响应：**
```json
{
  "success": true,
  "message": "已标记全部通知为已读"
}
```

### 清空通知接口

**请求：**
```http
DELETE /api/notifications/clear
```

**响应：**
```json
{
  "success": true,
  "message": "已清空所有通知"
}
```

## 📱 五、响应式规则

```
桌面端（>= 1024px）：
- 通知列表：完整显示
- 筛选器：横向布局
- 分页器：完整显示

平板端（768px - 1023px）：
- 通知列表：完整显示
- 筛选器：换行显示
- 分页器：简化显示

手机端（< 768px）：
- 通知列表：单列显示
- 筛选器：折叠显示
- 分页器：仅显示上一页/下一页
```
