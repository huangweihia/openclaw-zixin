## 📐 五、数据库表设计

### notifications 表

```sql
CREATE TABLE notifications (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '通知 ID',
    user_id BIGINT UNSIGNED NOT NULL COMMENT '用户 ID',
    type VARCHAR(50) NOT NULL COMMENT '通知类型',
    title VARCHAR(255) NOT NULL COMMENT '通知标题',
    content TEXT NULL COMMENT '通知内容',
    is_read TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否已读',
    read_at TIMESTAMP NULL COMMENT '已读时间',
    link_url VARCHAR(255) NULL COMMENT '关联链接',
    link_type VARCHAR(100) NULL COMMENT '关联类型',
    link_id BIGINT UNSIGNED NULL COMMENT '关联 ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_is_read (is_read),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='系统通知表';
```

**字段说明：**

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例 |
|------|------|------|--------|------|------|
| id | BIGINT | ✅ | 自增 | 通知 ID | 1 |
| user_id | BIGINT | ✅ | - | 用户 ID | 1 |
| type | VARCHAR | ✅ | - | 通知类型 | audit_approved |
| title | VARCHAR | ✅ | - | 通知标题 | 审核通过 |
| content | TEXT | ❌ | NULL | 通知内容 | 您的文章已通过审核 |
| is_read | TINYINT(1) | ✅ | 0 | 是否已读 | 0/1 |
| read_at | TIMESTAMP | ❌ | NULL | 已读时间 | 2026-04-03 10:00:00 |
| link_url | VARCHAR | ❌ | NULL | 关联链接 | /articles/123 |
| link_type | VARCHAR | ❌ | NULL | 关联类型 | App\Models\Article |
| link_id | BIGINT | ❌ | NULL | 关联 ID | 123 |

**type 枚举值说明：**

| 值 | 说明 | 触发场景 |
|----|------|---------|
| audit_approved | 审核通过 | 发布内容审核通过 |
| audit_rejected | 审核拒绝 | 发布内容审核拒绝 |
| like_received | 获得点赞 | 内容获得点赞 |
| comment_received | 获得评论 | 内容获得评论 |
| favorite_received | 获得收藏 | 内容获得收藏 |
| vip_expiring_soon | 会员到期提醒 | VIP 剩余 7 天 |
| vip_expired | 会员已过期 | VIP 已过期 |
| system_announcement | 系统公告 | 系统活动/更新 |

**业务规则：**
1. 删除用户时自动删除通知（ON DELETE CASCADE）
2. 仅保留最近 90 天的通知（定时清理）
3. 未读通知数超过 99 时显示 99+
4. 点击通知时自动标记为已读
5. 通知设置关闭时不推送对应类型通知

### user_notification_settings 表

```sql
CREATE TABLE user_notification_settings (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '设置 ID',
    user_id BIGINT UNSIGNED NOT NULL UNIQUE COMMENT '用户 ID',
    audit_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '审核通知开关',
    interaction_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '互动通知开关',
    vip_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '会员通知开关',
    system_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '系统公告开关',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户通知设置表';
```

**字段说明：**

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例 |
|------|------|------|--------|------|------|
| id | BIGINT | ✅ | 自增 | 设置 ID | 1 |
| user_id | BIGINT | ✅ | - | 用户 ID（唯一） | 1 |
| audit_enabled | TINYINT(1) | ✅ | 1 | 审核通知开关 | 0/1 |
| interaction_enabled | TINYINT(1) | ✅ | 1 | 互动通知开关 | 0/1 |
| vip_enabled | TINYINT(1) | ✅ | 1 | 会员通知开关 | 0/1 |
| system_enabled | TINYINT(1) | ✅ | 1 | 系统公告开关 | 0/1 |

**业务规则：**
1. 每个用户只能有一条设置记录（user_id 唯一）
2. 注册时自动创建，默认全部开启
3. 删除用户时自动删除设置（ON DELETE CASCADE）
