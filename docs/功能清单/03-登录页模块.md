## 功能 5：登录按钮

| 项目 | 内容 |
|------|------|
| **功能描述** | 验证字段并提交登录 |
| **页面位置** | 登录页底部 |
| **涉及数据库** | 表：users<br>字段：users.email, users.password, users.last_login_at, users.last_login_ip |
| **交互详情** | <table><tr><th>元素</th><th>点击/悬停</th><th>反应</th><th>跳转/动作</th></tr><tr><td>**登录**</td><td>点击</td><td>验证字段</td><td>POST /login</td></tr><tr><td>**登录（失败）**</td><td>-</td><td>显示错误</td><td>"邮箱或密码错误"</td></tr><tr><td>**登录（成功）**</td><td>-</td><td>显示"登录中..."</td><td>默认跳转首页；若带 **`return`**（站内相对路径，如 `/payments/confirm?plan=vip`）则跳转该地址</td></tr></table> |
| **API 调用** | POST /login<br>Body: { email, password, remember, return? }；亦支持 **GET /login?return=** 写入隐藏域 |
| **数据逻辑** | 1. 验证邮箱格式<br>2. 验证密码必填<br>3. 查询 users.email<br>4. bcrypt 验证密码<br>5. 创建 Session<br>6. 如果 remember=true，设置 remember_token<br>7. 更新 last_login_at 和 last_login_ip<br>8. **`return` 校验**：仅允许以 `/` 开头且非 `//` 的相对路径（`LoginController::safeInternalPath`） |
| **开发状态** | ✅ 已完成 |
| **测试状态** | ⬜ 未测试 |
