## ⚙️ 与 `_laravel_temp` 实现对照（2026-04-05）

| 清单项 | 当前代码实现 |
|--------|----------------|
| 确认页 | **`GET /payments/confirm?plan=`**（`PaymentController@confirm`）；视图 **`resources/views/payments/confirm.blade.php`**（`layouts.site`）；**未登录**展示「登录后下单」并带 **`return` 回跳**；免费方案不创建订单 |
| 结果页 | **`resources/views/payments/result.blade.php`**（演示支付按钮受 **`PAYMENT_SIMULATE`** 控制） |
| 创建订单 | **`POST /payments/orders`**（需登录），写入 **`orders`**（`Order` 模型），跳转 **`GET /payments/result?order_no=`** |
| 支付结果页 | **`GET /payments/result`**，仅可查看本人订单 |
| 演示支付 | **`POST /payments/orders/{order}/simulate-paid`**；**`config/openclaw.php`** → **`PAYMENT_SIMULATE`**（`.env`），为 **false** 时路由 **404**；成功后将订单标 **paid**，并更新 **`users.role` / `subscription_ends_at`**（**`admin` 角色不改 role**，仅延长到期） |
| 微信真实支付 / 回调 | **未实现**（仍为 ⏸️） |

## 功能 1：支付确认页

| 项目 | 内容 |
|------|------|
| **功能描述** | 展示订单信息并确认支付 |
| **页面位置** | /payments/confirm |
| **涉及数据库** | 表：orders（创建订单在提交后）<br>字段：orders.order_no, orders.amount, orders.status |
| **交互详情** | 已登录且为 VIP/SVIP 方案：**「创建订单并支付」** → POST 下单；未登录：**登录后下单**（带回跳） |
| **数据逻辑** | 校验 plan；生成 **order_no**；金额见 **`VipController::planAmountYuan`**；**product_id**：1=vip，2=svip |
| **开发状态** | ✅ **已完成（Web 演示闭环）** |
| **测试状态** | ⬜ 未测试 ⬜ 测试中 ⬜ 测试通过（代码侧有 **`tests/Feature/PaymentDemoTest.php`**） |
