# OpenClaw 智信 - 用户发布模块

> **版本：** v4.0  
> **创建时间：** 2026-04-03  
> **模块类型：** UGC 内容生态核心  
> **功能总数：** 19 个（见第七节统计表）

---

## ⚙️ 与 `_laravel_temp` 实现对照（2026-04-04）

| 能力 | 清单规格 | 当前代码 |
|------|----------|----------|
| 投稿创建 | 富文本、封面、多标签等 | **MVP**：`POST /dashboard/posts`，字段 `type`（case/tool/experience/resource/question）、`title`、`content`、`visibility`；**无** 独立 `/api/publish` |
| 我的投稿 | 卡片列表、筛选、编辑 | **`GET /dashboard/posts`** 表格分页列表；**无** 状态 Tab 筛选、编辑/删除 |
| 审核 | 后台审核台 | **`GET /admin/user-posts`**（需 `users.role=admin`），`POST .../approve`、`POST .../reject`（`audit_note` 必填） |
| 对外展示 | 首页/频道展示 UGC | **`GET /posts`** 列表 + **`GET /posts/{id}`** 详情；仅 **`status=approved`** 且 **`visibility` 为 public / vip** 进列表；**private** 不进广场，作者可打开详情；VIP 文与站内文章一致为摘要 + 开通引导 |
| 通知 | 审核结果通知 | **`notifications` 表 + 审核通过/拒绝写入；`/notifications` 通知中心；顶栏未读角标**（非文档中的 `/api/notifications`） |

**数据库表名：** `user_posts`（非文档第九节示例中的 `publishes` 表名，以迁移为准）。

## 📐 九、数据库表设计

### publishes 表

```sql
CREATE TABLE publishes (
    id BIGINT UNSIGNED PRIMARY KEY,
    user_id BIGINT UNSIGNED NOT NULL,
    type ENUM('case', 'tool', 'resource', 'experience') NOT NULL,
    title VARCHAR(200) NOT NULL,
    cover_images JSON,
    content TEXT NOT NULL,
    tags JSON,
    extra_fields JSON,
    status ENUM('pending', 'approved', 'rejected') DEFAULT 'pending',
    reject_reason TEXT NULL,
    suggest TEXT NULL,
    views_count INT DEFAULT 0,
    likes_count INT DEFAULT 0,
    favorites_count INT DEFAULT 0,
    comments_count INT DEFAULT 0,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```
