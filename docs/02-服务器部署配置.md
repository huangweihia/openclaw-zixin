# OpenClaw 智信 - 服务器部署配置（Docker Compose）

> 适用对象：已将仓库代码拉取到 Linux 服务器，使用 `docker-compose.server.yml` 部署。  
> 说明：本文件是**服务器专用版**，与 `01-开发环境配置.md`（本地开发专用）分离。

---

## 1. 目录与启动入口

确保 Laravel 根目录中包含以下文件（同层）：

- `artisan`
- `docker/`
- `docker-compose.server.yml`
- `.env`

进入部署目录（示例）：

```bash
cd /opt/zixin/openclaw-zixin
```

启动服务：

```bash
docker compose -f docker-compose.server.yml up -d --build
docker compose -f docker-compose.server.yml ps
```

---

## 2. 生产环境变量（`.env`）

至少确保以下项正确：

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=http://你的域名或IP

DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=openclaw_zhixin
DB_USERNAME=ai_side
DB_PASSWORD=aiside123456

REDIS_HOST=redis
REDIS_PASSWORD=null
REDIS_PORT=6379
SESSION_DRIVER=redis
```

改完后清缓存：

```bash
docker compose -f docker-compose.server.yml exec -T php php artisan config:clear
docker compose -f docker-compose.server.yml exec -T php php artisan cache:clear
```

---

## 3. 依赖安装（服务器）

> 服务器不要使用 `npm run dev`，只使用 `npm run build`。

PHP 依赖：

```bash
docker compose -f docker-compose.server.yml exec -T php composer install --no-dev --optimize-autoloader --no-interaction
```

前端依赖与构建：

```bash
docker compose -f docker-compose.server.yml exec -T php npm install
docker compose -f docker-compose.server.yml exec -T php npm run build
docker compose -f docker-compose.server.yml exec -T php php artisan optimize:clear
```

---

## 4. 初始化 Laravel

```bash
docker compose -f docker-compose.server.yml exec -T php php artisan key:generate
docker compose -f docker-compose.server.yml exec -T php php artisan migrate --force
```

> **小程序登录**：若接口返回 `SQLSTATE[42S22]`、`Unknown column 'wechat_mini_openid'` 或 `wechat_openid`，说明数据库未执行到最新迁移，请确认已执行上文的 `migrate --force`，且部署中包含 `2026_04_17_120000_add_wechat_mini_fields_to_users_table`（小程序 openid/unionid）与 `2026_04_18_100000_add_wechat_openid_to_users_table`（独立 `wechat_openid` 字段并从旧列回填）。

可选：初始化种子数据（会写入默认管理员/演示数据）：

```bash
docker compose -f docker-compose.server.yml exec -T php php artisan db:seed --force
```

## 5. 数据库可视化连接说明

本项目默认将 MySQL 绑定为：

```text
127.0.0.1:3309 -> 容器3306
```

含义：

- 服务器本机可连接
- 外网默认不可直连（更安全）

Navicat / DBeaver 推荐使用 SSH 隧道：

- 常规页：`127.0.0.1:3309`
- SSH 页：服务器公网 IP + `22` + 登录用户（如 `ubuntu`）

1Panel 里需要“登记远程数据库”才会显示：

- 地址：`127.0.0.1`
- 端口：`3309`
- 用户：`ai_side`
- 密码：`aiside123456`

## 6. 常见问题

### 6.1 `vendor/autoload.php` 不存在

说明依赖未成功安装：

```bash
docker compose -f docker-compose.server.yml exec -T php composer install --no-dev --optimize-autoloader --no-interaction
docker compose -f docker-compose.server.yml restart php
```

### 6.2 连接数据库失败（`Connection refused`）

检查 `.env` 是否误写成 `127.0.0.1/root`，应改为 `mysql/ai_side`。

### 6.3 前端改动未生效

服务器执行：

```bash
docker compose -f docker-compose.server.yml exec -T php npm run build
docker compose -f docker-compose.server.yml exec -T php php artisan optimize:clear
```

## 7. 上线后日常更新

```bash
bash /opt/openclaw-zixin/scripts/server-update.sh
```

### 7.1 单行一键命令（不清空数据）

> 作用：拉取最新代码、执行迁移（仅增量，不重置数据）、更新前后端缓存并重启 PHP/Nginx。  
> 兼容：若 `php` 容器没有 `npm`，自动用 `node:20-alpine` 临时容器打包前端。

```bash
cd /opt/openclaw-zixin && git pull --ff-only origin main && docker compose -f docker-compose.server.yml up -d && docker compose -f docker-compose.server.yml exec -T php composer install --no-dev --optimize-autoloader --no-interaction && docker compose -f docker-compose.server.yml exec -T php php artisan migrate --force && (docker compose -f docker-compose.server.yml exec -T php sh -lc "command -v npm >/dev/null 2>&1 && npm ci && npm run build" || docker run --rm -v "$PWD:/app" -w /app node:20-alpine sh -lc "npm ci && npm run build") && docker compose -f docker-compose.server.yml exec -T php php artisan optimize:clear && docker compose -f docker-compose.server.yml exec -T php php artisan config:cache && docker compose -f docker-compose.server.yml exec -T php php artisan route:cache && docker compose -f docker-compose.server.yml exec -T php php artisan view:cache && docker compose -f docker-compose.server.yml restart php nginx
```

### 7.2 脚本方式（推荐）

仓库已内置脚本：`scripts/server-update.sh`

```bash
chmod +x /opt/openclaw-zixin/scripts/server-update.sh
bash /opt/openclaw-zixin/scripts/server-update.sh
```

## 8. 站内消息、站点公告与推送（网站机制 · 与小程序对齐）

### 8.1 站点公告（`announcements`）

- **后台**：Filament「公告管理」或旧管理端 `announcements` 资源；数据表 `announcements`。
- **官网前台**：`layouts/site`、`home` 等通过 Blade 片段 `partials/announcement-marquee`、`partials/announcement-float` 展示（`AppServiceProvider` 中 View::composer 按 `is_published`、过期时间、`display_position` / `is_floating` 筛选）。
- **小程序**：公开接口 `GET /api/public/announcements`（列表，支持 `floating`、`placement`、`limit` 查询参数）、`GET /api/public/announcements/{id}`（详情）；首页与「我的 → 站点公告」入口读取同一数据源。

### 8.2 用户站内信（`notifications` / 消息中心）

- **表**：`notifications`（模型 `InboxNotification`），与官网「通知中心」列表同源。
- **来源 A — 系统通知（`system_notifications`）**  
  - 后台创建/发布/更新后，由 `SystemNotificationInboxDispatcher::dispatchIfNeeded` 按 `audience`（all / member / vip / svip / admin / non_member 等）**批量插入** `notifications`，并写入 `system_notifications.inbox_dispatched_at` 防止重复派发。  
  - 管理端列表 `GET /api/admin/system-notifications` 会顺带执行 `dispatchBacklog` 兜底历史未派发记录。  
  - **非微信实时推送**：依赖上述写入；无单独「微信订阅消息」链路（若后续接入需在业务层再发）。
- **来源 B — 站内推送（`push_notifications`）**  
  - 旧 Vue 管理端 `PushNotificationController` 在创建/更新时调用 `PushNotificationInboxService::syncFromPush`，按 **单用户** `user_id` 在 `notifications` 中 upsert（存在 `push_notification_id` 列时关联）。若仅通过 Filament 维护 `push_notifications` 且未接同一服务，需自行保证与 `notifications` 一致（或复用上述 Service）。

### 8.3 调度与运维提示

- 当前 `app/Console/Kernel.php` **未**为系统通知单独注册定时任务；派发主要在 **后台保存/拉列表** 时触发。若长期不打开后台，可在服务器 cron 中增加自定义 Artisan 命令定期调用 `dispatchBacklog`（需自行加命令，本仓库可按需扩展）。
- 小程序「消息中心」调用 `GET /api/wechat/mini/inbox*`（需登录 Sanctum），与官网登录用户看到的站内信一致。
