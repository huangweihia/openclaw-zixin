# OpenClaw 智信 - 服务器部署配置（Docker Compose）

> 适用对象：已将仓库代码拉取到 Linux 服务器，使用 `docker-compose.server.yml` 部署。  
> 说明：本文件是**服务器专用版**，与 `01-开发环境配置.md`（本地开发专用）分离。

---

## 1. 目录与启动入口

确保 Laravel 根目录中包含以下文件（同层）：

- `artisan`
- `docker/`
- `docker-compose.server.yml`
- `.env`

进入部署目录（示例）：

```bash
cd /opt/zixin/openclaw-zixin
```

启动服务：

```bash
docker compose -f docker-compose.server.yml up -d --build
docker compose -f docker-compose.server.yml ps
```

---

## 2. 生产环境变量（`.env`）

至少确保以下项正确：

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=http://你的域名或IP

DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=openclaw_zhixin
DB_USERNAME=ai_side
DB_PASSWORD=aiside123456

REDIS_HOST=redis
REDIS_PASSWORD=null
REDIS_PORT=6379
SESSION_DRIVER=redis
```

改完后清缓存：

```bash
docker compose -f docker-compose.server.yml exec -T php php artisan config:clear
docker compose -f docker-compose.server.yml exec -T php php artisan cache:clear
```

---

## 3. 依赖安装（服务器）

> 服务器不要使用 `npm run dev`，只使用 `npm run build`。

PHP 依赖：

```bash
docker compose -f docker-compose.server.yml exec -T php composer install --no-dev --optimize-autoloader --no-interaction
```

前端依赖与构建：

```bash
docker compose -f docker-compose.server.yml exec -T php npm install
docker compose -f docker-compose.server.yml exec -T php npm run build
docker compose -f docker-compose.server.yml exec -T php php artisan optimize:clear
```

---

## 4. 初始化 Laravel

```bash
docker compose -f docker-compose.server.yml exec -T php php artisan key:generate
docker compose -f docker-compose.server.yml exec -T php php artisan migrate --force
```

可选：初始化种子数据（会写入默认管理员/演示数据）：

```bash
docker compose -f docker-compose.server.yml exec -T php php artisan db:seed --force
```

## 5. 数据库可视化连接说明

本项目默认将 MySQL 绑定为：

```text
127.0.0.1:3309 -> 容器3306
```

含义：

- 服务器本机可连接
- 外网默认不可直连（更安全）

Navicat / DBeaver 推荐使用 SSH 隧道：

- 常规页：`127.0.0.1:3309`
- SSH 页：服务器公网 IP + `22` + 登录用户（如 `ubuntu`）

1Panel 里需要“登记远程数据库”才会显示：

- 地址：`127.0.0.1`
- 端口：`3309`
- 用户：`ai_side`
- 密码：`aiside123456`

## 6. 常见问题

### 6.1 `vendor/autoload.php` 不存在

说明依赖未成功安装：

```bash
docker compose -f docker-compose.server.yml exec -T php composer install --no-dev --optimize-autoloader --no-interaction
docker compose -f docker-compose.server.yml restart php
```

### 6.2 连接数据库失败（`Connection refused`）

检查 `.env` 是否误写成 `127.0.0.1/root`，应改为 `mysql/ai_side`。

### 6.3 前端改动未生效

服务器执行：

```bash
docker compose -f docker-compose.server.yml exec -T php npm run build
docker compose -f docker-compose.server.yml exec -T php php artisan optimize:clear
```
