## 📋 一、开发流程总览

```
第 1 步：阅读功能清单 → 了解功能做什么
        ↓
第 2 步：查看组件设计文档 → 了解使用哪些组件
        ↓
第 3 步：查看数据库设计 → 了解字段结构
        ↓
第 4 步：编写后端接口 → 实现数据逻辑
        ↓
第 5 步：查看原型图 → 了解页面布局
        ↓
第 6 步：开发前端页面 → 对照原型图 1:1 还原
        ↓
第 7 步：自测功能闭环 → 确保能正常使用
        ↓
第 8 步：更新功能清单 → 标记为✅已完成
```

**⚠️ 核心原则：开发顺序不能颠倒！**

1. **功能描述** - 了解功能做什么
2. **涉及数据库** - 知道用哪些表和字段
3. **交互详情** - 理解交互逻辑
4. **数据逻辑** - 理解数据处理流程

**示例：**
```markdown
## 功能 6：实时动态滚动
**功能描述：** 实时动态自动滚动显示
**页面位置：** 首页实时动态区
**涉及数据库：** 表：users, subscriptions
              字段：users.name, users.role, subscriptions.plan
**交互详情：**
| 元素 | 点击/悬停 | 反应 | 动作 |
|------|----------|------|------|
| 实时动态 | 悬停 | 暂停滚动 | 无 |
| 实时动态 | 点击 | 无 | 无（纯展示）|
**数据逻辑：**
1. 查询最近 10 条用户开通 VIP 记录
2. 对用户手机号/邮箱脱敏处理
3. 每 5 秒自动滚动一条
```

→ **理解：** 需要在首页显示一个自动滚动的动态列表，数据来自 users 和 subscriptions 表

### 第 4 步：编写后端接口

**操作：**
1. 创建/编辑控制器
2. 实现数据逻辑
3. 添加路由

**示例：**
```php
// app/Http/Controllers/HomeController.php
public function index()
{
    // 数据逻辑：查询最近 10 条用户开通 VIP 记录
    $vipActivities = DB::table('subscriptions')
        ->join('users', 'subscriptions.user_id', '=', 'users.id')
        ->where('subscriptions.plan', '!=', 'free')
        ->orderByDesc('subscriptions.created_at')
        ->limit(10)
        ->get([
            'users.name',
            'users.role',
            'subscriptions.plan',
            'subscriptions.created_at'
        ]);
    
    // 脱敏处理
    $vipActivities->transform(function($item) {
        $item->name = Str::limit($item->name, 1, '***');
        return $item;
    });
    
    return view('home', compact('vipActivities'));
}

// routes/web.php
Route::get('/', [HomeController::class, 'index'])->name('home');
```

## 📝 六、常见问题

### Q1: 功能无法独立闭环怎么办？

**A:** 标记为"依赖 XXX 功能"，先开发依赖的功能。

**示例：**
```markdown
## 功能 10：VIP 专属内容
**依赖功能：** 功能 1-VIP 会员体系
**开发状态：** ⬜ 等待依赖功能完成
```

---

### Q2: 组件未开发怎么办？

**A:** 两种方案：
1. 先开发组件（推荐）
2. 临时用 HTML 替代，后续替换为组件

---

### Q3: 数据库表不存在怎么办？

**A:** 先创建 migration，执行迁移后再开发功能。

```bash
php artisan make:migration create_xxx_table
php artisan migrate
```

---

### Q4: 如何确认功能是否闭环？

**A:** 问自己三个问题：
1. 用户能看到什么？（页面展示）
2. 用户能做什么？（交互操作）
3. 数据从哪里来？（后端接口）

如果都能回答，说明功能闭环了。
