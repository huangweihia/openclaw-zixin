## 🧩 六、补充：后台管理员与权限（2026-04-14）

> 本节为后台管理端新增的权限与管理员拆分表结构说明。  
> 说明：后台登录仍基于 `users`（认证与密码沿用），但后台账号体系与权限配置独立维护在下列表中。

### 6.1 admin_users（后台管理员表）

**用途：** 将后台管理员从前台用户概念中抽出（便于独立启停、超级管理员标记、后续权限管理）。

```sql
CREATE TABLE admin_users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT UNSIGNED NOT NULL UNIQUE COMMENT '关联 users.id（认证沿用 users）',
  display_name VARCHAR(255) NULL COMMENT '后台显示名（可不同于 users.name）',
  is_active TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用（禁用后禁止后台登录）',
  is_super TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否超级管理员（拥有所有权限）',
  last_login_at TIMESTAMP NULL COMMENT '最后登录时间（后台）',
  last_login_ip VARCHAR(64) NULL COMMENT '最后登录 IP（后台）',
  created_at TIMESTAMP NULL,
  updated_at TIMESTAMP NULL,
  CONSTRAINT fk_admin_users_user_id FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='后台管理员表';
```

**关键规则：**
- 管理员首次登录后台时，系统会自动创建一条 `admin_users` 记录（兼容历史 admin 账号）。
- `is_active = 0` 时拒绝后台登录。
- `is_super = 1` 时默认拥有 `*` 全权限（RBAC 未配置也不会导致菜单消失）。

## 📋 二、表结构详细设计（40 个表）

### 2.1 用户相关表（6 个）

#### 表 1：users（用户表）⭐核心表

**用途：** 存储用户基本信息

**数据量预估：** 10,000+ 条

**读写比例：** 读 80% / 写 20%

```sql
CREATE TABLE users (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '用户 ID',
    name VARCHAR(255) NOT NULL COMMENT '用户昵称',
    email VARCHAR(255) NOT NULL UNIQUE COMMENT '邮箱（登录账号）',
    email_verified_at TIMESTAMP NULL COMMENT '邮箱验证时间',
    password VARCHAR(255) NOT NULL COMMENT '密码（bcrypt 加密）',
    avatar VARCHAR(255) NULL COMMENT '头像 URL',
    role ENUM('user', 'vip', 'svip', 'admin') NOT NULL DEFAULT 'user' COMMENT '用户角色',
    subscription_ends_at TIMESTAMP NULL COMMENT 'VIP 到期时间',
    last_login_at TIMESTAMP NULL COMMENT '最后登录时间',
    last_login_ip VARCHAR(45) NULL COMMENT '最后登录 IP',
    remember_token VARCHAR(100) NULL COMMENT '记住我 token',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_email (email),
    INDEX idx_role (role),
    INDEX idx_subscription_ends_at (subscription_ends_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

**字段详细说明：**

| 字段 | 类型 | 必填 | 默认值 | 长度 | 说明 | 示例 | 验证规则 | 前端处理 |
|------|------|------|--------|------|------|------|---------|---------|
| id | BIGINT UNSIGNED | ✅ | 自增 | - | 用户 ID（主键） | 1 | 只读 | 隐藏 |
| name | VARCHAR | ✅ | - | 255 | 用户昵称 | 张三 | 2-50 字符 | 输入框 |
| email | VARCHAR | ✅ | - | 255 | 邮箱（唯一，登录账号） | user@example.com | 邮箱格式，唯一 | 输入框 |
| email_verified_at | TIMESTAMP | ❌ | NULL | - | 邮箱验证时间 | 2026-04-03 10:00:00 | 只读 | 隐藏 |
| password | VARCHAR | ✅ | - | 255 | 密码（bcrypt 加密） | $2y$10$... | 最少 6 字符 | 密码框 |
| avatar | VARCHAR | ❌ | NULL | 255 | 头像 URL | /avatars/1.png | URL 格式 | 文件上传 |
| role | ENUM | ✅ | 'user' | - | 用户角色 | user/vip/svip/admin | 只能是枚举值 | 下拉框 |
| subscription_ends_at | TIMESTAMP | ❌ | NULL | - | VIP 到期时间 | 2027-04-03 10:00:00 | - | 只读 |
| last_login_at | TIMESTAMP | ❌ | NULL | - | 最后登录时间 | 2026-04-03 10:00:00 | - | 只读 |
| last_login_ip | VARCHAR | ❌ | NULL | 45 | 最后登录 IP | 192.168.1.1 | IP 格式 | 只读 |
| remember_token | VARCHAR | ❌ | NULL | 100 | 记住我 token | abc123... | - | 隐藏 |
| created_at | TIMESTAMP | ✅ | CURRENT | - | 创建时间 | 2026-04-03 10:00:00 | - | 只读 |
| updated_at | TIMESTAMP | ✅ | CURRENT | - | 更新时间 | 2026-04-03 10:00:00 | - | 只读 |

**role 枚举值说明：**

| 值 | 说明 | 权限 | 如何获得 | 前端标识 |
|----|------|------|---------|---------|
| user | 普通用户 | 基础功能 | 注册默认 | 无标识 |
| vip | VIP 会员 | VIP 内容 + 基础功能 | 购买 VIP | 👑 VIP 标识 |
| svip | SVIP 私教 | 全部功能 + 专属服务 | 购买 SVIP | 👑 SVIP 标识 |
| admin | 管理员 | 后台管理权限 | 管理员设置 | 🔧 后台入口 |

**业务规则：**
1. email 必须唯一，用于登录
2. password 必须使用 bcrypt 加密（Laravel Hash::make）
3. role 只能是：user, vip, svip, admin
4. 注册时自动设置 email_verified_at
5. 登录时自动更新 last_login_at 和 last_login_ip
6. VIP 到期后自动降级为 user（需要定时任务）
7. 密码最少 6 位，建议 8 位以上
8. 头像上传限制：JPG/PNG/GIF，最大 2MB

**关联表：**
- user_posts（用户发布）
- favorites（收藏）
- view_histories（浏览历史）
- subscriptions（订阅）
- orders（订单）
- comments（评论）

**PC 端展示：**
- 用户列表：表格展示，显示头像、昵称、邮箱、角色、注册时间
- 用户详情：卡片展示，显示完整信息

**手机端展示：**
- 用户列表：列表展示，显示头像、昵称、角色
- 用户详情：卡片展示，显示关键信息

#### 表 2：user_posts（用户发布表）

**用途：** 存储用户发布的内容（副业案例、工具推荐、经验心得等）

**数据量预估：** 5,000+ 条

**读写比例：** 读 70% / 写 30%

```sql
CREATE TABLE user_posts (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '发布 ID',
    user_id BIGINT UNSIGNED NOT NULL COMMENT '发布用户 ID',
    type ENUM('case', 'tool', 'experience', 'resource', 'question') NOT NULL COMMENT '发布类型',
    title VARCHAR(255) NOT NULL COMMENT '标题',
    content TEXT NOT NULL COMMENT '内容（Markdown）',
    category VARCHAR(100) NULL COMMENT '分类',
    tags JSON NULL COMMENT '标签（JSON 数组）',
    cover_image VARCHAR(255) NULL COMMENT '封面图 URL',
    attachments JSON NULL COMMENT '附件（JSON 数组）',
    visibility ENUM('public', 'vip', 'private') NOT NULL DEFAULT 'public' COMMENT '可见性',
    status ENUM('pending', 'approved', 'rejected') NOT NULL DEFAULT 'pending' COMMENT '审核状态',
    audit_note TEXT NULL COMMENT '审核备注',
    audited_by BIGINT UNSIGNED NULL COMMENT '审核人 ID',
    audited_at TIMESTAMP NULL COMMENT '审核时间',
    view_count INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '浏览数',
    like_count INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '点赞数',
    comment_count INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '评论数',
    favorite_count INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '收藏数',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (audited_by) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_user_id (user_id),
    INDEX idx_type (type),
    INDEX idx_status (status),
    INDEX idx_visibility (visibility),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户发布表';

**type 枚举值说明：**

| 值 | 说明 | 审核要求 | VIP 限制 | 字数要求 | 图标 |
|----|------|---------|---------|---------|------|

| experience | 经验心得 | 需审核 | 无 | 最少 200 字 | 💡 |
| resource | 资源分享 | 需审核 | 仅 VIP | 最少 100 字 | 📦 |
| question | 问答求助 | 自动通过 | 无 | 最少 50 字 | ❓ |

**visibility 枚举值说明：**

| 值 | 说明 | 谁可以看到 | 图标 |
|----|------|-----------|------|
| public | 公开 | 所有人（包括未登录） | 🌍 |
| vip | VIP 专属 | VIP 及以上用户 | 👑 |
| private | 私密 | 仅自己和管理员 | 🔒 |

**status 枚举值说明：**

| 值 | 说明 | 操作 | 通知 | 前端标识 |
|----|------|------|------|---------|
| pending | 待审核 | 等待管理员审核 | 无 | 🟡 审核中 |
| approved | 已通过 | 公开展示 | 发送通过通知 | 🟢 已通过 |
| rejected | 已拒绝 | 不公开，通知用户 | 发送拒绝通知（含原因） | 🔴 已拒绝 |

**业务规则：**
1. 用户只能删除自己的发布
2. 发布后状态默认为 pending（待审核）
3. 管理员审核后状态变为 approved 或 rejected
4. 审核通过后根据 visibility 决定是否公开显示
5. 资源分享类型仅 VIP 用户可以发布
6. 删除用户时自动删除其所有发布（ON DELETE CASCADE）
7. 审核人删除时设置为 NULL（ON DELETE SET NULL）
8. 封面图限制：JPG/PNG/GIF，最大 2MB
9. 附件限制：PDF/DOC/DOCX，单个最大 10MB，最多 5 个

**关联表：**
- users（用户）
- comments（评论）
- favorites（收藏）

**PC 端展示：**
- 列表页：卡片式布局，每行 3 个，显示封面、标题、摘要、统计
- 详情页：完整内容，左侧内容，右侧相关信息

**手机端展示：**
- 列表页：单列卡片，显示封面、标题、摘要
- 详情页：单列布局，内容自适应

    WHERE is_active = 1 
    AND user_id IN (SELECT id FROM users WHERE vip_level = 'svip')

# 2. 对每个订阅执行检索
for sub in subscriptions:
    try:
        # 调用 MCP 工具检索
        items = mcp.search_by_keywords(
            keywords=sub['keywords'],
            sources=sub['sources'],
            exclude=sub['exclude_keywords'],
            limit=50
        )
        
        # 推送给用户
        if items:
            push_to_user(
                user_id=sub['user_id'],
                methods=sub['push_methods'],
                items=items,
                subscription_name=sub['name']
            )
            
            # 更新执行记录
            db.update('svip_custom_subscriptions', sub['id'], {
                'last_run_at': datetime.now(),
                'last_items_count': len(items)
            })
            
            # 记录日志
            db.insert('svip_subscription_logs', {
                'subscription_id': sub['id'],
                'items_found': len(items),
                'items_pushed': len(items),
                'status': 'success',
                'execution_time_ms': execution_time
            })
    except Exception as e:
        # 记录失败日志
        db.insert('svip_subscription_logs', {
            'subscription_id': sub['id'],
            'status': 'failed',
            'error_message': str(e),
            'execution_time_ms': execution_time
        })
```

**OpenClaw 调用示例：**
```python
# 1. 读取所有启用的订阅
subscriptions = db.query("""
    SELECT * FROM svip_custom_subscriptions 
    WHERE is_active = 1 
    AND user_id IN (SELECT id FROM users WHERE vip_level = 'svip')
""")

# 2. 对每个订阅执行检索
for sub in subscriptions:
    try:
        # 调用 MCP 工具检索
        items = mcp.search_by_keywords(
            keywords=sub['keywords'],
            sources=sub['sources'],
            exclude=sub['exclude_keywords'],
            limit=50
        )
        
        # 推送给用户
        if items:
            push_to_user(
                user_id=sub['user_id'],
                methods=sub['push_methods'],
                items=items,
                subscription_name=sub['name']
            )
            
            # 更新执行记录
            db.update('svip_custom_subscriptions', sub['id'], {
                'last_run_at': datetime.now(),
                'last_items_count': len(items)
            })
            
            # 记录日志
            db.insert('svip_subscription_logs', {
                'subscription_id': sub['id'],
                'items_found': len(items),
                'items_pushed': len(items),
                'status': 'success',
                'execution_time_ms': execution_time
            })
    except Exception as e:
        # 记录失败日志
        db.insert('svip_subscription_logs', {
            'subscription_id': sub['id'],
            'status': 'failed',
            'error_message': str(e),
            'execution_time_ms': execution_time
        })
```

**数据量预估：** 5,000+ 条（与用户数 1:1）

**读写比例：** 读 90% / 写 10%

```sql
CREATE TABLE user_notification_settings (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '设置 ID',
    user_id BIGINT UNSIGNED NOT NULL UNIQUE COMMENT '用户 ID',
    audit_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '审核通知开关',
    interaction_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '互动通知开关',
    vip_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '会员通知开关',
    system_enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '系统公告开关',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户通知设置表';
```

**字段详细说明：**

| 字段 | 类型 | 必填 | 默认值 | 长度 | 说明 | 示例 | 验证规则 | 前端处理 |
|------|------|------|--------|------|------|------|---------|---------|
| id | BIGINT UNSIGNED | ✅ | 自增 | - | 设置 ID | 1 | 只读 | 隐藏 |
| user_id | BIGINT UNSIGNED | ✅ | - | - | 用户 ID（唯一） | 1 | 必须存在，唯一 | 自动获取 |
| audit_enabled | TINYINT(1) | ✅ | 1 | 1 | 审核通知开关 | 0/1 | 0 或 1 | 开关 |
| interaction_enabled | TINYINT(1) | ✅ | 1 | 1 | 互动通知开关 | 0/1 | 0 或 1 | 开关 |
| vip_enabled | TINYINT(1) | ✅ | 1 | 1 | 会员通知开关 | 0/1 | 0 或 1 | 开关 |
| system_enabled | TINYINT(1) | ✅ | 1 | 1 | 系统公告开关 | 0/1 | 0 或 1 | 开关 |
| created_at | TIMESTAMP | ✅ | CURRENT | - | 创建时间 | 2026-04-03 10:00:00 | - | 只读 |
| updated_at | TIMESTAMP | ✅ | CURRENT | - | 更新时间 | 2026-04-03 10:00:00 | - | 只读 |

**业务规则：**
1. 每个用户只能有一条设置记录（user_id 唯一）
2. 注册时自动创建，默认全部开启
3. 删除用户时自动删除设置（ON DELETE CASCADE）
4. 修改设置后立即生效

**关联表：**
- users（用户）- 外键关联

#### 表 35：points（积分流水表）

**用途：** 记录用户积分获取和消费流水

**数据量预估：** 50,000+ 条

**读写比例：** 读 60% / 写 40%

```sql
CREATE TABLE points (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '流水 ID',
    user_id BIGINT UNSIGNED NOT NULL COMMENT '用户 ID',
    amount INT NOT NULL COMMENT '积分变动（正数增加，负数减少）',
    balance INT UNSIGNED NOT NULL COMMENT '变动后余额',
    type ENUM('earn', 'spend') NOT NULL COMMENT '类型',
    category VARCHAR(50) NOT NULL COMMENT '分类',
    description VARCHAR(255) NOT NULL COMMENT '描述',
    reference_type VARCHAR(255) NULL COMMENT '关联类型',
    reference_id BIGINT UNSIGNED NULL COMMENT '关联 ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_type (type),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='积分流水表';
```

**字段详细说明：**

| 字段 | 类型 | 必填 | 默认值 | 长度 | 说明 | 示例 | 验证规则 | 前端处理 |
|------|------|------|--------|------|------|------|---------|---------|
| id | BIGINT UNSIGNED | ✅ | 自增 | - | 流水 ID | 1 | 只读 | 隐藏 |
| user_id | BIGINT UNSIGNED | ✅ | - | - | 用户 ID | 1 | 必须存在 | 自动获取 |
| amount | INT | ✅ | - | - | 积分变动 | +10/-50 | 正数或负数 | 只读 |
| balance | INT UNSIGNED | ✅ | - | - | 变动后余额 | 1234 | >= 0 | 只读 |
| type | ENUM | ✅ | - | - | 类型 | earn/spend | 只能是枚举值 | 只读 |
| category | VARCHAR | ✅ | - | 50 | 分类 | publish/like/comment | - | 只读 |
| description | VARCHAR | ✅ | - | 255 | 描述 | 发布文章奖励 | 10-255 字符 | 只读 |
| reference_type | VARCHAR | ❌ | NULL | 255 | 关联类型 | App\Models\UserPost | - | 只读 |
| reference_id | BIGINT UNSIGNED | ❌ | NULL | - | 关联 ID | 123 | 必须存在 | 只读 |
| created_at | TIMESTAMP | ✅ | CURRENT | - | 创建时间 | 2026-04-03 10:00:00 | - | 只读 |

**type 枚举值说明：**

| 值 | 说明 | amount 符号 | 场景 |
|----|------|-----------|------|
| earn | 赚取 | 正数 | 发布文章、签到、邀请 |
| spend | 消费 | 负数 | 兑换 VIP、下载资源 |

**category 分类说明：**

| 分类 | 类型 | 积分 | 说明 |
|------|------|------|------|
| publish | earn | +10 | 发布成功 |
| audit_approved | earn | +20 | 审核通过 |
| featured | earn | +100 | 精选内容 |
| like_received | earn | +1 | 获得点赞 |
| comment_received | earn | +2 | 获得评论 |
| vip_exchange | spend | -500 | 兑换 VIP 天 |
| resource_download | spend | -5 | 下载资源 |

**业务规则：**
1. 每次积分变动自动创建流水记录
2. amount 正数表示增加，负数表示减少
3. balance 是变动后的总余额
4. 删除用户时自动删除流水记录（ON DELETE CASCADE）
5. 积分永久有效，不过期

**关联表：**
- users（用户）- 外键关联

#### 表 36：publish_audits（发布审核表）

**用途：** 存储用户发布内容的审核记录

**数据量预估：** 10,000+ 条

**读写比例：** 读 70% / 写 30%

```sql
CREATE TABLE publish_audits (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '审核 ID',
    publish_id BIGINT UNSIGNED NOT NULL COMMENT '发布 ID',
    user_id BIGINT UNSIGNED NOT NULL COMMENT '用户 ID',
    auditor_id BIGINT UNSIGNED NULL COMMENT '审核人 ID',
    status ENUM('pending', 'approved', 'rejected') NOT NULL DEFAULT 'pending' COMMENT '审核状态',
    reject_reason TEXT NULL COMMENT '拒绝原因',
    suggest TEXT NULL COMMENT '修改建议',
    priority INT NOT NULL DEFAULT 0 COMMENT '审核优先级',
    audited_at TIMESTAMP NULL COMMENT '审核时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (publish_id) REFERENCES user_posts(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (auditor_id) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_publish_id (publish_id),
    INDEX idx_user_id (user_id),
    INDEX idx_auditor_id (auditor_id),
    INDEX idx_status (status),
    INDEX idx_priority (priority)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='发布审核表';
```

**字段详细说明：**

| 字段 | 类型 | 必填 | 默认值 | 长度 | 说明 | 示例 | 验证规则 | 前端处理 |
|------|------|------|--------|------|------|------|---------|---------|
| id | BIGINT UNSIGNED | ✅ | 自增 | - | 审核 ID | 1 | 只读 | 隐藏 |
| publish_id | BIGINT UNSIGNED | ✅ | - | - | 发布 ID | 1 | 必须存在 | 只读 |
| user_id | BIGINT UNSIGNED | ✅ | - | - | 用户 ID | 1 | 必须存在 | 自动获取 |
| auditor_id | BIGINT UNSIGNED | ❌ | NULL | - | 审核人 ID | 2 | 必须存在 | 只读 |
| status | ENUM | ✅ | 'pending' | - | 审核状态 | pending/approved/rejected | 只能是枚举值 | 只读 |
| reject_reason | TEXT | ❌ | NULL | - | 拒绝原因 | 内容过于简单 | rejected 时必填 | 文本域 |
| suggest | TEXT | ❌ | NULL | - | 修改建议 | 建议补充更多细节 | 可选 | 文本域 |
| priority | INT | ✅ | 0 | - | 审核优先级 | 0/10/20 | 数值越大越优先 | 只读 |
| audited_at | TIMESTAMP | ❌ | NULL | - | 审核时间 | 2026-04-03 10:00:00 | - | 只读 |
| created_at | TIMESTAMP | ✅ | CURRENT | - | 创建时间 | 2026-04-03 10:00:00 | - | 只读 |
| updated_at | TIMESTAMP | ✅ | CURRENT | - | 更新时间 | 2026-04-03 10:00:00 | - | 只读 |

**status 枚举值说明：**

| 值 | 说明 | 操作 | 通知用户 |
|----|------|------|---------|
| pending | 待审核 | 等待管理员审核 | 无 |
| approved | 已通过 | 内容上线 | 发送通过通知 |
| rejected | 已拒绝 | 内容不公开 | 发送拒绝通知（含原因） |

**priority 优先级规则：**

| 用户等级 | 优先级值 | 说明 |
|---------|---------|------|
| 免费用户 | 0 | 普通优先级 |
| VIP 用户 | 10 | 优先审核 |
| SVIP 用户 | 20 | 最优先审核 |

**业务规则：**
1. 用户发布内容时自动创建审核记录
2. status 默认为 pending（待审核）
3. 管理员审核后状态变为 approved 或 rejected
4. priority 根据发布者会员等级自动设置
5. 删除发布时自动删除审核记录（ON DELETE CASCADE）
6. 审核人删除时设置为 NULL（ON DELETE SET NULL）

**关联表：**
- user_posts（发布内容）- 外键关联
- users（用户）- 外键关联
